    Alpha-research diagnostics

    The signal-evaluation half of the research SDK: information coefficient, alpha decay, signal half-life, and capacity analysis. These are the questions a quant asks before committing capital to a signal — how well the alpha predicts forward returns, at what horizon that power lives, how fast the signal mean-reverts, and how much notional it can absorb before market impact eats the edge.

    Like muninn.factor, this module is numpy + pandas only and self-contained — nothing here imports the HTTP client, so every function runs on any returns/​signal arrays.

    Worked example

    examples/ic_capacity_research.py demonstrates ic_decay_curve, signal_half_life, and the rest of this module on an offline synthetic panel.

    muninn.research.ic(signal, forward_return, *, method='spearman', horizon=1)

    Information coefficient of a signal vs a forward-return series.

    Computes the (rank or linear) correlation plus its significance and a directional hit-rate, packaged in an :class:ICResult. The sign of the IC is the sign of the signal's predictive relationship: a signal that is high before up-moves has a positive IC.

    Parameters:

    Name Type Description Default
    signal ArrayLike

    Alpha score per bar (1-D, time-ordered).

    		 required
    forward_return ArrayLike

    Forward return per bar, same length and alignment as signal (e.g. from :func:forward_returns).

    		 required
    method str

    "spearman" (default, rank IC — robust to outliers) or "pearson" (linear IC).

    		 'spearman'
    horizon int

    Recorded on the result for bookkeeping; does not change the maths (the caller is responsible for passing a forward return of that horizon).

    		 1

    muninn.research.rank_ic(signal, forward_return, *, horizon=1)

    Convenience: the Spearman (rank) IC value alone.

    muninn.research.ic_decay_curve(prices, signal, *, horizons=(1, 5, 15), method='spearman', log_returns=False)

    IC at a range of forward horizons — the alpha-decay curve.

    For each horizon h it computes the forward return over h bars from prices (via :func:forward_returns) and the IC of signal against it. The result tabulates how predictive power changes as the horizon grows: a fast-decaying microstructure alpha peaks at h=1 and fades, while a slower factor signal may strengthen out to longer horizons.

    Parameters:

    Name Type Description Default
    prices ArrayLike

    Price series used to build forward returns at each horizon.

    		 required
    signal ArrayLike

    Alpha score per bar, aligned to prices.

    		 required
    horizons Iterable[int]

    Iterable of positive bar horizons to evaluate.

    		 (1, 5, 15)
    method str

    "spearman" or "pearson" (passed to :func:ic).

    		 'spearman'
    log_returns bool

    Use log forward returns if True.

    		 False

    Returns:

    Type Description
    DataFrame

    One row per horizon with columns ["horizon", "ic", "t_stat", "p_value", "hit_rate", "n"], sorted ascending by horizon.

    muninn.research.ICResult dataclass

    One horizon's information-coefficient diagnostics.

    Attributes:

    Name Type Description
    horizon int

    Forward horizon in bars.
    ic float

    The correlation between the signal and the forward return (rank/Spearman or Pearson per the call).
    t_stat float

    IC t-statistic ic * sqrt((n - 2) / (1 - ic^2)) under the null of zero correlation.
    p_value float

    Two-sided p-value of t_stat against Student-t with n - 2 dof.
    hit_rate float

    Fraction of observations where sign(signal) == sign(return) (directional agreement), over the non-nan, non-zero-signal pairs.
    n int

    Number of aligned (non-nan) observations used.
    method str

    "spearman" or "pearson".

    muninn.research.forward_returns(prices, *, horizon=1, log=False)

    N-bar forward return aligned to the current bar.

    out[t] = price[t+horizon] / price[t] - 1 (or the log thereof). The last horizon entries are nan because their forward observation is past the end of the series. This is the numpy sibling of :func:muninn.notebook.forward_returns (which operates on a Polars frame).

    Parameters:

    Name Type Description Default
    prices ArrayLike

    1-D price series, time-ordered ascending.

    		 required
    horizon int

    Number of bars ahead. Must be a positive integer.

    		 1
    log bool

    If True return log(p_{t+h}/p_t) instead of the simple return.

    		 False

    muninn.research.autocorrelation(signal, *, max_lag=20)

    Sample autocorrelation function of a signal for lags 0..max_lag.

    Uses the standard biased estimator (divide by n, not n - k) so the ACF is positive semidefinite and lag 0 is exactly 1. nan entries are dropped before estimation.

    Returns:

    Type Description
    ndarray

    Array of length max_lag + 1; element k is the lag-k autocorrelation.

    muninn.research.signal_half_life(signal)

    AR(1) mean-reversion half-life of a signal, in bars.

    Fits the AR(1) coefficient phi by regressing x_t on x_{t-1} (the lag-1 autocorrelation for a demeaned series) and converts it to the half-life -ln(2) / ln(phi) — the number of bars for a shock to decay to half its size. A signal that persists (phi near 1) has a long half-life; one that mean-reverts quickly has a short one.

    Returns:

    Type Description
    float

    Half-life in bars. inf if the signal does not mean-revert (phi >= 1); nan if phi <= 0 (no exponential interpretation) or the signal is degenerate.

    muninn.research.capacity_estimate(*, gross_sharpe, adv_notional, impact_coefficient, trades_per_year, max_sharpe_erosion=0.1, volatility_per_trade=0.01)

    Max deployable notional under a square-root market-impact model.

    The square-root law models the per-trade price impact of trading a participation rate q = notional / adv_notional as

    .. math:: \text{impact} = \kappa \,\sigma\, \sqrt{q} = \kappa \,\sigma\, \sqrt{\frac{N}{\text{ADV}}},

    where kappa is the (dimensionless) impact_coefficient and sigma the per-trade volatility. Impact is a cost deducted from each trade's gross edge. The gross edge per trade is recovered from the strategy's annualised gross Sharpe as

    .. math:: \text{edge} = \frac{\text{Sharpe}_\text{gross}\,\sigma} {\sqrt{\text{trades_per_year}}},

    (Sharpe = mean / vol annualised ⇒ per-trade mean = Sharpe·σ/√T). We allow impact to erode at most max_sharpe_erosion of that edge; setting impact = erosion · edge and solving for N gives the closed-form capacity

    .. math:: N^* = \text{ADV}\, \left(\frac{\text{erosion}\cdot\text{edge}}{\kappa\,\sigma}\right)^2 .

    Because capacity scales as 1 / kappa^2, raising the impact coefficient lowers capacity — the core property the test pins down.

    Parameters:

    Name Type Description Default
    gross_sharpe float

    Annualised gross (pre-impact) Sharpe ratio of the strategy. Must be positive (a strategy with no edge has no capacity to speak of).

    		 required
    adv_notional float

    Average daily volume in notional terms — the liquidity denominator in the participation rate. Must be positive.

    		 required
    impact_coefficient float

    kappa in the square-root law. Larger = more impact per unit participation = less capacity. Must be positive.

    		 required
    trades_per_year float

    Number of trades per year, used to convert annualised Sharpe to a per-trade edge. Must be positive.

    		 required
    max_sharpe_erosion float

    Fraction of the per-trade edge you tolerate losing to impact (e.g. 0.10 = give up at most 10%). In (0, 1].

    		 0.1
    volatility_per_trade float

    Per-trade return volatility sigma. Must be positive.

    		 0.01

    Returns:

    Type Description
    CapacityResult

    muninn.research.CapacityResult dataclass

    Result of a square-root-impact capacity estimate.

    Attributes:

    Name Type Description
    capacity_notional float

    Maximum per-trade notional (same currency as adv_notional) that keeps the Sharpe erosion at or below max_sharpe_erosion.
    impact_at_capacity float

    Per-trade impact (as a return fraction) incurred at that notional.
    gross_edge float

    The gross per-trade edge implied by gross_sharpe and trades_per_year (return fraction per trade) — the budget that impact is allowed to erode.
    binding_erosion float

    The Sharpe-erosion fraction actually achieved at capacity (equals max_sharpe_erosion when the impact constraint binds).
